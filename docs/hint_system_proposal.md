# Hangman 힌트 시스템 제안 요약 문서  
*(Based on Issue #46, 행맨 게임 힌트 시스템 추가 제안)*

이 문서는 과거에 제안했던 **Hangman 게임 힌트 시스템** 추가 아이디어를
문서의 형태로 남겨두기 위해 작성되었습니다.  
당시 프로젝트 내에서는 힌트 기능이 존재하지 않았으며,  
게임의 난이도가 높아 사용자 경험이 떨어질 수 있다는 점을 문제로 인식했습니다.

본 문서는 **당시 제안했던 근거, 배경, 제안 내용**을 기록하는 용도로 사용됩니다.

---

## 1. 제안 배경

당시 Hangman 게임은 다음과 같은 구조적 특징이 있었습니다:

- 단어 난이도에 따라 난이도가 급격하게 변동
- 실패 횟수가 누적되면 난이도를 조정할 수 있는 장치 부족
- 사용자들이 중간에 게임을 포기할 가능성 있음

특히 거의 모든 글자를 맞혀도 마지막 하나에서 실패하는 경우가 잦아,
플레이 체감 난이도가 불필요하게 높아지는 문제가 있었습니다.

따라서 플레이어가 중도 포기하지 않고 끝까지 진행할 수 있도록  
**힌트(hint) 기능을 추가하는 것을 제안**하게 되었습니다.

---

## 2. 제안 당시 이슈에서 제안했던 내용

### (1) 힌트 버튼을 통해 글자 힌트 제공  
- 플레이어가 원할 때 `힌트` 버튼을 눌러  
  **아직 공개되지 않은 글자 중 하나를 자동 공개**하도록 제안했습니다.

### (2) 실패 횟수 기준 힌트 자동 제공  
- 실패 횟수가 일정 기준(예: 3회 이상)을 넘으면  
  **자동으로 힌트를 지급**하도록 제안했습니다.

### (3) 힌트 사용 제한  
- 난이도 조절을 위해 다음과 같은 제약을 둘 것을 제안했습니다:
  - 힌트는 게임당 1~2회만 사용 가능  
  - 힌트 사용 시 점수 또는 보상에 영향을 줄 수 있음  

---

## 3. 제안의 목적

이 기능을 제안한 핵심 목적은 다음과 같습니다:

- **사용자 경험(User Experience) 개선**
- **게임 몰입도 유지**
- **게임 난이도를 부드럽게 조절**
- **플레이어의 중도 포기 감소**

미니게임에서는 지나치게 높은 난이도보다 지속적인 즐거움과 접근성이 중요하므로  
힌트 시스템은 사용성 향상에 큰 도움이 된다고 판단했습니다.

---

## 4. 향후 구현 또는 개선 시 참고할 수 있는 추가 아이디어

- 난이도(Easy/Normal/Hard)별 힌트 정책 분리  
- 힌트 사용 시 효과음 또는 시각적 강조 효과  
- 힌트 사용 시 점수/점수 페널티 시스템  
- 랜덤 힌트 대신 사용자의 진행 상황을 고려한 스마트 힌트 시스템  

---

## 5. 제안 당시 결론

이 문서는 단순히 힌트 기능을 설명하는 가이드가 아니라,  
**당시에 프로젝트 품질 향상을 위해 제출했던 제안의 논리적 근거와 방향성**을 남긴 기록입니다.

향후 기여자들이  
- 이 기능이 왜 필요했는지  
- 어떤 방향으로 제안되었는지  
- 어떤 배경에서 결정되었는지  

를 파악하는 데 도움이 될 것이라고 생각합니다.
