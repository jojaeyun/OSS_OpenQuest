# 프로젝트 폴더 구조 개선 제안 요약  
*(Based on Issue #45, 메인 브랜치 기능별 폴더화 제안)*

이 문서는 과거에 제안했던 **기능별 폴더 구조 개편 제안**을 정리해 남기는 문서입니다.  
당시 프로젝트는 모든 파일(`.py`, `.png`, `.mp3`, `.wav`)이 **루트 디렉토리에 혼재**되어 있었고,  
이로 인해 유지보수성과 가독성이 매우 낮은 상태였습니다.

해당 제안은 프로젝트의 구조적 안정성과 협업 효율성을 향상시키기 위한 목적에서 제출되었습니다.

---

## 1. 제안 배경

과거에는:

- 파이썬 코드, 이미지, 사운드 파일이 모두 루트에 저장됨
- 어떤 파일이 어떤 게임에 속하는지 식별하기 어려움
- 코드 리뷰나 기능 추가 시 범위 파악이 어려움
- 협업하는 사람마다 파일 배치 기준이 달라 충돌 위험 증가

이러한 이유로 **폴더화가 반드시 필요하다고 판단**했습니다.

---

## 2. 당시 제안한 주요 개선안

### (1) 게임별로 완전히 독립된 폴더 생성
예시 제안 형태:

<game_name>/
├── assets/
├── sounds/
├── images/
└── main_game_file.py

### (2) 리소스(assets) 분리  
이미지, 사운드, 폰트 등은 모두 각 게임 폴더 안의  
`assets/`, `sound/` 폴더에 분리하여 저장할 것을 제안했습니다.

### (3) 공용 자원 분리  
여러 게임에서 사용되는 폰트나 공통 라이브러리는  
`fonts/` 등의 공용 폴더로 분리할 것을 제안했습니다.

### (4) 새로운 게임 추가 시 따라야 할 기본 구조 정의  
하나의 템플릿 구조를 제공하여  
새 게임을 추가하는 사람들이 동일한 규칙을 따르도록 제안했습니다.

---

## 3. 제안의 목적

- **프로젝트 전체 가독성 향상**
- **협업 시 수정 범위 명확화**
- **파일 충돌 최소화**
- **새 기여자가 진입하기 쉬운 구조 제공**
- **유지보수성 강화를 통한 프로젝트 수명 연장**

당시 저장소의 구조를 비교적 빠르게 정리할 수 있었고,  
추후 기능 추가에도 큰 도움이 될 것이라고 판단했습니다.

---

## 4. 향후 개선 시 참고 사항

- 게임별 README 템플릿 제공  
- 공용 asset 규칙 가이드 작성  
- 새 게임 추가 시 자동으로 구조 생성하는 스크립트 작성(선택)

---

## 5. 제안 당시 결론

이 문서는 폴더 구조 개편 제안의 **배경과 기술적 근거를 남기기 위한 기록**입니다.  
프로젝트가 확장될수록 구조 정리의 중요성은 커지기 때문에,  
향후 기여자들이 같은 기준을 유지하도록 돕기 위해 작성되었습니다.
